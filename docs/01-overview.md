# 01 - 项目概述

## 项目名称

**Owl** — macOS 系统健康实时监控 Menu Bar App

## 项目起源

在一次 macOS 性能排查中，我们发现：

1. PM2 进程管理器的 crash-loop 导致 `launchservicesd` CPU 飙到 107%，但没有任何直观的告警机制
2. Microsoft Defender 异常扫描占用 70-92% CPU，用户完全无感知
3. CPU 散热节流、进程卡死、USB 设备异常等问题，全部静默发生

这些问题的共同特征是：**macOS 统一日志系统（Unified Logging）已经记录了所有异常信号**，但没有工具将这些信号实时呈现给用户。

## 核心目标

在 macOS Menu Bar 上提供一个常驻的系统健康指示器：

1. **实时监控** — 通过分析 `log stream` 输出，检测已知的异常 pattern
2. **零延迟感知** — 异常发生时，Menu Bar 图标立即变色/变形，用户第一时间发现
3. **极低开销** — 监控本身不能成为性能瓶颈，目标 CPU < 1%，内存 < 30MB
4. **可操作** — 每个告警附带简短的说明和建议操作，而非只给原始日志

## 设计原则

- **被动观察，不主动干预** — 只读取日志流，不修改系统配置
- **内核侧预过滤** — 通过 predicate 让 log 子系统在内核侧就过滤掉无关日志，减少到达 app 的数据量
- **简单算法** — 只用阈值检测、速率计数、状态追踪三种模型，不引入 ML 或统计分析
- **模块化检测器** — 每个 pattern 是独立的检测器模块，可独立启用/禁用/调参

## 目标用户

- macOS 高级用户 / 开发者
- 需要了解系统健康状态但不想一直开着 Activity Monitor 的人
- 在企业环境中使用 Mac，受安全软件影响但无法自行卸载的用户

## 非目标

- 不是 Activity Monitor 的替代品（不展示进程列表/CPU 曲线）
- 不是日志查看器（不展示原始日志内容）
- 不是系统优化工具（不自动清理/修复）
- 不上架 Mac App Store（需要非沙箱权限）
